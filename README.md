# 🛠️ Система уведомлений о техническом обслуживании оборудования АСУТП

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
![Platform: Windows](https://img.shields.io/badge/platform-Windows-green.svg)
![Version](https://img.shields.io/badge/version-2.6.1-brightgreen.svg)

Автоматизированная система мониторинга и уведомлений о сроках технического обслуживания оборудования АСУТП (Автоматизированные системы управления технологическими процессами) с веб-интерфейсом для управления задачами обслуживания и ведением журнала выполненных работ.

![Пример email-уведомления](data/mail_screenshot.png)

## 📋 Описание

Данная система предназначена для:
- **Веб-интерфейса** для мониторинга и управления задачами обслуживания в реальном времени
- Автоматического отслеживания сроков технического обслуживания оборудования
- Формирования и отправки уведомлений по электронной почте
- Визуализации статистики обслуживания с интерактивными графиками
- Ведения истории выполненных работ и автоматического создания резервных копий

Система анализирует данные из Excel-таблицы и автоматически отправляет уведомления о:
- 🚨 **Срочном обслуживании** - оборудование требует немедленного внимания
- ⚠️ **Приближающемся обслуживании** - сроки подходят к концу
- ✅ **Нормальном состоянии** - все оборудование обслужено вовремя

## 🚀 Установка и настройка

### Требования
- Python 3.8 или выше
- Windows OS (для работы с Excel и Task Scheduler)
- Microsoft Excel (для пересчета формул через xlwings)
- Доступ к SMTP серверу (для email уведомлений)
- Веб-браузер (для доступа к дашборду)

### Установка зависимостей

```bash
# Установка из requirements.txt (рекомендуется)
pip install -r requirements.txt

# Основные пакеты:
pip install flask pandas matplotlib xlwings openpyxl
```

**ВАЖНО:** Система использует xlwings для пересчета формул Excel и openpyxl для быстрых операций записи.

### Установка в виртуальном окружении

```bash
# Создание виртуального окружения
python -m venv .venv

# Активация (Windows)
.venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt
```

### Настройка Excel-таблицы

Таблица должна содержать следующие колонки:
1. №
2. Объект
3. Наименование
4. Обозначение
5. Место расположения
6. Работы (ранее "Выполнить")
7. Интервал ТО (дней)
8. Напоминание (за дней)
9. Дата последнего ТО
10. Дата следующего ТО (автоматически рассчитывается)
11. Статус (автоматически определяется)

## 🌐 Запуск веб-интерфейса

### Локальный запуск

```bash
# Активируйте виртуальное окружение
.venv\Scripts\activate

# Запустите веб-сервер
python maintenance_web.py
```

Приложение будет доступно по адресу:
- **Локально:** `http://localhost:5940`
- **Из сети:** `http://<IP-сервера>:5940`

### Развертывание в локальной сети

1. **Настройте брандмауэр Windows:**
   - Откройте порт **5940** для входящих подключений TCP
   - Windows Defender Firewall → Правила для входящих подключений → Создать правило

2. **Запустите приложение:**
   ```bash
   python maintenance_web.py
   ```

3. **Доступ с других ПК:**
   - Узнайте IP-адрес сервера: `ipconfig`
   - Откройте браузер на любом ПК в сети: `http://10.100.56.X:5940`

### Запуск как Windows Service (для автозапуска)

Используйте **NSSM** (Non-Sucking Service Manager):

```bash
# Скачайте NSSM с nssm.cc
nssm install MaintenanceWebService

# В окне NSSM укажите:
# Path: C:\path\to\.venv\Scripts\python.exe
# Startup directory: C:\path\to\maintenance_alert
# Arguments: maintenance_web.py

# Запустите сервис
nssm start MaintenanceWebService
```

## ⚙️ Автоматические Email-уведомления

### Настройка Windows Task Scheduler

Для автоматического ежедневного запуска:

1. **Откройте Планировщик заданий**
   - Пуск → Планировщик заданий
   - Или выполните `taskschd.msc`

2. **Создайте новую задачу**
   - Действия → Создать задачу...
   - Имя: `Maintenance Alert ASUTP`
   - Описание: `Ежедневная проверка оборудования АСУТП`

3. **Настройте триггер**
   - Вкладка "Триггеры" → Создать
   - Ежедневно в 07:00
   - Повторять каждый день

4. **Настройте действие**
   - Вкладка "Действия" → Создать
   - Программа: `python.exe`
   - Аргументы: `maintenance_alert.py`
   - Рабочая папка: путь к папке проекта

5. **Дополнительные настройки**
   - Вкладка "Условия": снимите галочки питания
   - Вкладка "Параметры": разрешите принудительный запуск

⚠️ Внимание  **Создайте папки:**
Когда задача запускается без опции "Выполнять только при наличии пользователя", Windows запускает её в изолированной сессии (Session 0), где нет профиля пользователя, нет рабочего стола, и Excel не может нормально инициализировать своё окружение (включая временные папки, настройки, COM-объекты и т.п.).
Если вы обязаны запускать задачу без пользователя (например, сервер без GUI), можно вручную создать недостающие системные папки, которые Excel ожидает найти.

```cmd
mkdir "C:\Windows\System32\config\systemprofile\Desktop"
mkdir "C:\Windows\SysWOW64\config\systemprofile\Desktop"
```

## 📊 Формат данных

### Статусы оборудования
- 🚨 **ОБСЛУЖИТЬ** - требуется немедленное обслуживание (просрочено)
- ⚠️ **Внимание** - приближается срок обслуживания (в пределах напоминания)
- ✅ **Не требуется** - оборудование обслужено вовремя

### Журнал обслуженного оборудования

Система автоматически ведёт журнал всех фактов обслуживания оборудования:
- **Мгновенная запись** при отметке через веб-интерфейс
- **Автоматическое обнаружение** при изменении дат ТО в Excel (scheduled run)
- **Дедупликация** по комбинации (лист, номер строки, дата) — исключает двойные записи
- **Автоматическая очистка** записей старше 62 дней
- **Синхронизация снимков** — обновление `excel_snapshot.json` после ручных отметок предотвращает дубли при следующем плановом запуске

#### Формат записи журнала:
```json
{
  "date": "2026-05-03",
  "timestamp": "2026-05-03T14:32:15.123456",
  "sheet": "ПК АСУ ТП",
  "row": 42,
  "designation": "ШКА-101",
  "name": "Шкаф управления насосом",
  "object": "Цех №1"
}
```

### Структура Excel-таблицы

**Листы:** `ПК АСУ ТП` и `Шкафы АСУ ТП`
**Диапазон данных:** A4:K300

| Колонка | Название | Описание |
|---------|----------|----------|
| A | № | Порядковый номер |
| B | Объект | Название объекта/системы |
| C | Наименование | Модель оборудования |
| D | Обозначение | Уникальный идентификатор |
| E | Место расположения | Физическое расположение |
| F | Работы | Действия для выполнения |
| G | Интервал ТО (дней) | Периодичность обслуживания |
| H | Напоминание (за дней) | За сколько дней предупреждать |
| I | Дата последнего ТО | Когда проводилось ТО |
| J | Дата следующего ТО | Автоматически рассчитывается |
| K | Статус | Автоматически определяется |

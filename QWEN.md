# Система уведомлений о техническом обслуживании оборудования АСУТП

## 📋 Project Overview

**Maintenance Alert System** — это автоматизированная система мониторинга и уведомлений о сроках технического обслуживания оборудования АСУТП (Автоматизированные системы управления технологическими процессами).

**Текущая версия:** 2.5.1 (дата выпуска: 10.01.2026)

### Основное назначение
- **Ежедневная автоматическая проверка** сроков ТО оборудования
- **Email-уведомления** о срочном обслуживании с цветовой кодировкой статусов
- **Веб-интерфейс** для мониторинга и управления задачами обслуживания в реальном времени
- **Отметка обслуженного оборудования** через браузер с автоматическим обновлением Excel
- **Визуализация статистики** с интерактивными графиками
- **Автоматическое резервное копирование** данных перед изменениями

### Ключевые компоненты

| Файл | Описание |
|------|----------|
| `maintenance_alert.py` | Основной скрипт (backend логика, проверка сроков, отправка email) |
| `maintenance_web.py` | Веб-приложение Flask (дашборд, фильтрация, отметка ТО) |
| `Обслуживание ПК и шкафов АСУТП.xlsx` | Excel-таблица с данными оборудования |
| `requirements.txt` | Зависимости Python |

### Технологии
- **Python 3.8+**
- **Flask 3.1+** — веб-фреймворк
- **pandas** — работа с данными Excel
- **matplotlib** — визуализация статистики
- **xlwings** — пересчет формул Excel (COM-интеграция)
- **openpyxl** — быстрые операции записи Excel
- **Windows Task Scheduler** — автоматический запуск

## 🚀 Building and Running

### Установка зависимостей

```bash
# Создание виртуального окружения (рекомендуется)
python -m venv .venv

# Активация (Windows)
.venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt
```

### Запуск компонентов

#### 1. Консольный режим (ежедневная проверка)
```bash
python maintenance_alert.py
```
**Что делает:**
- Читает Excel-файл, принудительно пересчитывает все формулы через xlwings
- Анализирует сроки ТО по всем позициям
- Определяет статусы: 🚨 ОБСЛУЖИТЬ, ⚠️ Внимание, ✅ Не требуется
- Отправляет email-уведомления получателям
- Обновляет статистику в `data/maintenance_alert_history.json`
- Создаёт резервную копию Excel перед изменениями

#### 2. Веб-интерфейс (мониторинг и управление)
```bash
python maintenance_web.py
```
**Доступ:**
- Локально: `http://localhost:5940`
- Из сети: `http://<IP-сервера>:5940`

**Возможности:**
- Интерактивный дашборд со статусами оборудования
- Фильтрация по объекту, статусу, обозначению, типу листа
- Отметка "Обслужено" с обновлением Excel
- Массовое обслуживание (checkbox + кнопка)
- Скачивание отчётов (полная таблица / только срочные)
- Статистика с графиками за 62 дня

### Автоматический запуск (Production)

#### Windows Task Scheduler (для email-уведомлений)
Задача `maintenance_alert.py` уже создана на файловом сервере АСУ ТП:
- **Расписание:** Ежедневно в 07:00
- **Скрипт:** `Y:\Обслуживание оборудования АСУТП\maintenance_alert.py`
- **Batch-файл:** `run_maintenance_alert.bat` (для надёжного запуска)

#### Windows Service (для веб-интерфейса)
```bash
# Установка службы через NSSM
nssm.exe install MaintenanceWeb

# Настройка (в GUI):
# Path: C:\path\to\.venv\Scripts\python.exe
# Startup directory: Y:\Обслуживание оборудования АСУТП
# Arguments: maintenance_web.py

# Запуск
nssm.exe start MaintenanceWeb
```

**Требования:**
- Открыть порт **5940** в брандмауэре Windows
- Статический IP в сети 10.100.56.x

## 🔧 Development Conventions

### Код-стайл
- **PEP 8** — базовый стиль кода Python
- **Type hints** — используются в основных функциях (`typing.Dict`, `List`, `Optional`)
- **Docstrings** — обязательны для классов и публичных методов
- **Именование:**
  - Классы: `PascalCase` (например, `ExcelHandler`, `MaintenanceChecker`)
  - Функции/переменные: `snake_case`
  - Константы: `UPPER_CASE`

### Архитектурные паттерны
- **Классы-конфигурации:** `Config` хранит все константы и настройки
- **Разделение ответственности:**
  - `ExcelHandler` — чтение/запись Excel
  - `MaintenanceChecker` — логика определения статусов
  - `StatisticsManager` — управление историей и статистикой
  - `ReportGenerator` — создание отчётов и графиков
  - `EmailSender` — отправка уведомлений
- **DualLogger** — логирование в консоль и файл одновременно

### Практики тестирования
- **Ручное тестирование** через запуск скрипта в консоли
- **Тестирование веб-интерфейса** через браузер
- **Проверка резервных копий** в `backups_excel/`
- **Верификация записи** по времени модификации файлов

### Управление версиями
- **Semantic Versioning:** `MAJOR.MINOR.PATCH` (например, 2.5.1)
- **CHANGELOG** ведётся в `README.md` в разделе "История версий"
- **Git flow:**
  - `main` — стабильная версия
  - Feature-ветки для новых функций

## 📊 Data Format

### Excel-таблица
**Файл:** `Обслуживание ПК и шкафов АСУТП.xlsx`
**Листы:** `ПК АСУ ТП`, `Шкафы АСУ ТП`
**Диапазон данных:** `A4:K300`

| Колонка | Название | Тип | Описание |
|---------|----------|-----|----------|
| A | № | int | Порядковый номер |
| B | Объект | str | Название объекта (ЗИФ-1, ЗИФ-2, ...) |
| C | Наименование | str | Модель оборудования |
| D | Обозначение | str | Уникальный идентификатор |
| E | Место расположения | str | Физическое расположение |
| F | Работы | str | Действия для выполнения |
| G | Интервал ТО (дней) | int | Периодичность обслуживания |
| H | Напоминание (за дней) | int | За сколько дней предупреждать |
| I | Дата последнего ТО | date | Когда проводилось ТО |
| J | Дата следующего ТО | date | `=I2+G2` (автоматически) |
| K | Статус | str | `=ЕСЛИ(J2<=СЕГОДНЯ();"ОБСЛУЖИТЬ";...)` |

### Статусы оборудования
- 🚨 **ОБСЛУЖИТЬ** — просрочено (дата следующего ТО ≤ сегодня)
- ⚠️ **Внимание** — скоро срок (дата следующего ТО - сегодня ≤ напоминание)
- ✅ **Не требуется** — всё в порядке

### JSON-история (`maintenance_alert_history.json`)

## 🔍 Key Workflows

### 1. Ежедневная проверка (автоматически)
```
Task Scheduler (07:00)
    ↓
maintenance_alert.py
    ↓
xlwings: пересчёт формул Excel
    ↓
Анализ статусов по всем позициям
    ↓
Генерация email с цветовой кодировкой
    ↓
Отправка получателям (SMTP)
    ↓
Обновление statistics JSON
    ↓
Логирование в data/maintenance_alert.log
```

### 2. Отметка ТО через веб-интерфейс
```
Пользователь нажимает "Обслужено" в дашборде
    ↓
Flask: POST /mark_serviced (или /mark_serviced_batch)
    ↓
Проверка блокировки файла Excel
    ↓
Создание резервной копии в backups_excel/
    ↓
openpyxl: запись текущей даты в колонку "Дата последнего ТО"
    ↓
xlwings: пересчёт формул
    ↓
Верификация записи (время модификации)
    ↓
Обновление statistics JSON
    ↓
Редирект на дашборд с ?serviced_status=success
```

### 3. Генерация отчёта для скачивания
```
Пользователь нажимает иконку Excel
    ↓
Flask: GET /download_full или /download_maintenance
    ↓
Копирование Excel в tmp/
    ↓
Генерация отфильтрованной таблицы (если нужно)
    ↓
send_file() с автоскачиванием
    ↓
Очистка tmp/ (опционально)
```

## 🛠 Troubleshooting

### Файл Excel заблокирован
**Симптом:** Ошибка записи или предупреждение в email
**Решение:** Закрыть файл во всех приложениях перед запуском скрипта

### Веб-интерфейс недоступен из сети
**Проверка:**
```bash
# Проверка службы
nssm.exe status MaintenanceWeb

# Проверка порта
netstat -an | findstr 5940

# Проверка брандмауэра
Get-NetFirewallRule | Where-Object DisplayName -like "*Maintenance Web*"
```

### Ошибка отправки email
**Причины:**
- Недоступен SMTP-сервер `mgd-ex1.pavlik-gold.ru`
- Блокировка портов брандмауэром
- Изменение настроек SMTP

**Проверка:**
```bash
telnet mgd-ex1.pavlik-gold.ru 25
```

### xlwings не работает в Task Scheduler
**Решение:** Создать папки для сессии Session 0:
```cmd
mkdir "C:\Windows\System32\config\systemprofile\Desktop"
mkdir "C:\Windows\SysWOW64\config\systemprofile\Desktop"
```
